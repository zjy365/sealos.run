适用范围：仅私有云。

如果 Harbor 的域名是 `harbor.xxxx.com`，可以通过一个简单的 patch 镜像，把对应的 `hosts.toml` 下发到节点。

## 操作步骤

### 1. 在 master 节点创建目录

```bash
mkdir -p patch-harbor-xxx
cd patch-harbor-xxx
```

### 2. 编写 install.sh

将下面内容保存为 `install.sh`：

```bash
#!/bin/bash
DOMAIN=harbor.xxxx.com
mkdir -p "/etc/containerd/certs.d/${DOMAIN}"
cat > /etc/containerd/certs.d/${DOMAIN}/hosts.toml <<EOF
server = "https://${DOMAIN}"

[host."https://${DOMAIN}"]
  capabilities = ["pull", "resolve", "push"]
  skip_verify = true
EOF
```

注意：

- 将 `DOMAIN=harbor.xxxx.com` 修改为真实 Harbor 域名

### 3. 编写 Kubefile

将下面内容保存为 `Kubefile`：

```dockerfile
FROM scratch
LABEL sealos.io.type="patch"
COPY install.sh harbor-fix.sh
CMD ["bash harbor-fix.sh"]
```

### 4. 构建并执行 patch

编写好以上两个文件后，执行：

```bash
cd patch-harbor-xxx
sealos build -t harbor-fix:20260210 .
sealos run harbor-fix:20260210
```

## 说明

- 该配置会在 `/etc/containerd/certs.d/<DOMAIN>/hosts.toml` 下生成 Harbor 的 containerd 仓库配置
- `capabilities = ["pull", "resolve", "push"]` 表示允许拉取、解析和推送
- `skip_verify = true` 表示跳过 TLS 证书校验，适用于自签名证书场景，但不适合长期作为高安全配置

## 1. 一键导入

### 通用地址

```bash
curl https://gist.githubusercontent.com/cuisongliu/4fd5533c7674a5e5389b6cd5a9016aee/raw/b7520106dbc362359fe2f8ca7e0d2bf29dbc914b/cert-skip.sh | bash -s -- --mode auto "https://192.168.10.70.nip.io:443"
```

### 国内地址

```bash
curl https://objectstorageapi.hzh.sealos.run/paxilf30-static/cert.sh | bash -s -- --mode auto "https://192.168.10.70.nip.io:443"
```

## 2. Windows 系统证书导入

使用方法：

- 将 `https://192.168.10.70.nip.io` 替换为实际域名

### Auto 模式

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\wincert.ps1 -Url https://192.168.10.70.nip.io/
```

### 强制从 HTTP 拉取 CA

```powershell
.\wincert.ps1 -Mode http -Url https://192.168.10.70.nip.io/
```

### 强制通过 TLS 抓取服务端证书

```powershell
.\wincert.ps1 -Mode tls -Url https://192.168.10.70.nip.io
```

## 3. Mac 与 Linux 证书导入

暂时无法在飞书文档外展示此内容。

使用方法：

- 将 `https://192.168.10.70.nip.io` 替换为实际域名

先赋予脚本执行权限：

```bash
chmod +x ./mac-linux.sh
```

### auto 模式

默认先通过 HTTP 拉取 CA，再通过 TLS 抓取服务端证书：

```bash
./mac-linux.sh --mode auto https://192.168.10.70.nip.io/
```

### http 模式

强制从 `http://HOST:32000/ssl/ca.crt` 拉取证书：

```bash
./mac-linux.sh --mode http https://192.168.10.70.nip.io/
```

### tls 模式

强制使用 `openssl` 握手抓取服务器证书：

```bash
./mac-linux.sh --mode tls https://192.168.10.70.nip.io/
```

## 4. 验证是否生效（Mac）

使用 `curl` 验证：

```bash
curl -Iv https://192.168.10.70.nip.io/
```

如果信任成功，通常不会再看到 `self signed certificate` 相关错误。

## 怎么理解这些单价

### CPU

CPU 费用取决于你给应用或数据库分配了多少核，它更适合衡量持续计算型负载，例如在线应用、任务处理或数据库实例。

### 内存

内存费用和分配的 GiB 数直接相关。对 Java、Node.js、向量检索、缓存等内存敏感型负载，这一项通常不能只看峰值，要按稳定运行的资源规格去估算。

### 存储卷

存储卷更接近“资源保留成本”。即使计算资源暂停，只要持久化卷仍然保留，就可能继续产生相关存储费用。

### 端口

端口是公开访问入口的成本项之一。对外提供 HTTP、HTTPS 或其他公网协议服务时，要考虑端口数量对费用的影响。

### 网络

同一集群、同一区域内的内部访问通常不收取外网流量费用；对外提供服务时，外网流量按实际出网量计费。

## 什么是对象存储

对象存储可以理解为 Sealos 上的文件对象入口。它的核心目标不是替代数据库或本地卷，而是把“文件上传、权限控制、访问域名和容量统计”标准化。

如果你的资源需要以文件形式保存、访问或分发，通常就更适合进入对象存储。

## 典型使用路径

1. 创建 Bucket
2. 配置权限和访问策略
3. 上传文件或目录
4. 绑定访问域名和 HTTPS
5. 持续观察容量和流量

查看部署教程：[部署对象存储](/docs/getting-started/create-object-storage)


## 常见问题

### 什么时候适合公开访问

如果文件需要直接给浏览器、移动端或外部下载链接访问，通常才适合评估公开访问。否则优先保持私有。

### 对象存储和本地卷怎么选

如果你的目标是保存文件对象、附件、素材和下载资源，更适合对象存储；如果你的目标是给单个运行中的服务挂一个读写目录，更适合卷。

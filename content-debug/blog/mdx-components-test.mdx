## 嵌套的复杂示例

### Kubernetes 部署最佳实践

1. **准备工作**

   <Callout title="环境检查">
     在开始部署前，请确保： - 所有节点已安装 Docker - 网络互通且防火墙规则正确 -
     系统时间已同步
   </Callout>

2. **安装集群**

   ```bash
   # 初始化主节点
   sealos run labring/kubernetes:v1.25.0 --masters 192.168.0.2

   # 添加工作节点
   sealos add --nodes 192.168.0.3,192.168.0.4
   ```

3. **验证安装**

   | 检查项   | 命令                  | 预期结果         |
   | -------- | --------------------- | ---------------- |
   | 节点状态 | `kubectl get nodes`   | 所有节点 Ready   |
   | Pod 状态 | `kubectl get pods -A` | 所有 Pod Running |
   | 服务状态 | `kubectl get svc -A`  | 服务正常暴露     |

4. **常见问题**

   <Callout type="warning" title="节点无法加入集群">
     如果节点无法加入集群，请检查： - 网络连接是否正常 - Token 是否过期 -
     防火墙规则是否正确
   </Callout>

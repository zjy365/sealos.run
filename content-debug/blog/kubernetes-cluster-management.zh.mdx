# Kubernetes 集群管理：从入门到生产实践完整指南

在云原生时代，Kubernetes 已经成为容器编排的事实标准。本文将带你从基础概念到生产实践，全面了解 Kubernetes 集群管理的方方面面。

## 什么是 Kubernetes 集群

Kubernetes 集群由一组运行容器化应用程序的节点组成。每个集群至少包含一个工作节点和一个控制平面。

### 核心组件

- **Control Plane（控制平面）**：管理集群的决策中心
- **Worker Nodes（工作节点）**：运行容器化应用
- **etcd**：存储集群状态的键值数据库
- **kube-apiserver**：集群的 API 入口
- **kube-scheduler**：负责 Pod 调度
- **kube-controller-manager**：运行控制器进程

## 集群搭建最佳实践

### 1. 使用 Sealos 快速部署

使用 Sealos，你可以在几分钟内完成生产级 Kubernetes 集群的部署：

```bash
# 安装 Sealos
curl -sfL https://raw.githubusercontent.com/labring/sealos/main/scripts/install.sh | sh

# 部署 Kubernetes 集群
sealos run labring/kubernetes:v1.28.0 --masters 192.168.0.2,192.168.0.3,192.168.0.4 \
  --nodes 192.168.0.5,192.168.0.6,192.168.0.7 --passwd your-password
```

### 2. 高可用架构设计

生产环境建议：

- 3 个或以上奇数个 Master 节点
- 使用负载均衡器分发 API Server 流量
- etcd 独立部署或使用托管服务

## 集群管理工具

### kubectl 常用命令

```bash
# 查看集群信息
kubectl cluster-info

# 查看节点状态
kubectl get nodes

# 查看所有 Pod
kubectl get pods --all-namespaces

# 查看集群事件
kubectl get events --sort-by=.metadata.creationTimestamp
```

## 安全加固建议

### 1. RBAC 权限控制

```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  namespace: default
  name: pod-reader
rules:
  - apiGroups: ['']
    resources: ['pods']
    verbs: ['get', 'watch', 'list']
```

### 2. Network Policy

使用网络策略隔离不同环境的 Pod：

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: deny-all
spec:
  podSelector: {}
  policyTypes:
    - Ingress
    - Egress
```

## 性能优化

### 1. 资源限制

为每个容器设置合理的资源请求和限制：

```yaml
resources:
  requests:
    memory: '64Mi'
    cpu: '250m'
  limits:
    memory: '128Mi'
    cpu: '500m'
```

### 2. 水平自动扩缩容

使用 HPA 根据负载自动调整 Pod 数量：

```bash
kubectl autoscale deployment myapp --cpu-percent=50 --min=1 --max=10
```

## 故障排查技巧

### 常见问题诊断

1. **Pod 无法启动**

   ```bash
   kubectl describe pod <pod-name>
   kubectl logs <pod-name>
   ```

2. **节点状态异常**

   ```bash
   kubectl describe node <node-name>
   journalctl -u kubelet -f
   ```

3. **网络连通性问题**
   ```bash
   kubectl run tmp-shell --rm -i --tty --image nicolaka/netshoot -- /bin/bash
   ```

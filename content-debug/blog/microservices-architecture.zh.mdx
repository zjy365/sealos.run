# 微服务架构设计与实践：从单体到分布式

微服务架构已经成为构建大型分布式系统的主流方案。本文将分享微服务架构设计的核心原则和实践经验。

## 什么是微服务架构

微服务架构是一种将应用程序构建为一组小型、独立服务的方法，每个服务运行在自己的进程中，并通过轻量级机制（通常是 HTTP API）进行通信。

### 核心特征

- **单一职责**：每个服务专注于一个业务能力
- **独立部署**：可以独立更新和扩展
- **去中心化**：去中心化的数据管理和治理
- **容错设计**：系统能够优雅地处理失败

```typescript
// 使用 RabbitMQ 发布订单创建事件
import amqp from 'amqplib';

async function publishOrderCreated(order: Order) {
  const connection = await amqp.connect('amqp://rabbitmq');
```

## 分布式事务

### Saga 模式

```typescript
// 订单创建 Saga
class OrderCreationSaga {
  async execute(orderData: OrderData) {
    const sagaId = generateId();

    try {
      // 1. 创建订单
      const order = await orderService.createOrder(orderData);
      await sagaLog.record(sagaId, 'order_created', order);

      // 2. 预留库存
      await inventoryService.reserveItems(order.items);
      await sagaLog.record(sagaId, 'inventory_reserved');

      // 3. 处理支付
      await paymentService.processPayment(order.id, order.amount);
      await sagaLog.record(sagaId, 'payment_processed');

      // 4. 确认订单
      await orderService.confirmOrder(order.id);

      return order;
    } catch (error) {
      // 执行补偿事务
      await this.compensate(sagaId);
      throw error;
    }
  }

  async compensate(sagaId: string) {
    const logs = await sagaLog.getLogs(sagaId);

    // 倒序执行补偿操作
    for (const log of logs.reverse()) {
      if (log.action === 'payment_processed') {
        await paymentService.refund(log.data.orderId);
      }
      if (log.action === 'inventory_reserved') {
        await inventoryService.releaseItems(log.data.items);
      }
      if (log.action === 'order_created') {
        await orderService.cancelOrder(log.data.orderId);
      }
    }
  }
}
```

### 重试机制

```typescript
async function retryWithBackoff<T>(
  fn: () => Promise<T>,
  maxRetries = 3,
  delay = 1000,
): Promise<T> {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn();
    } catch (error) {
      if (i === maxRetries - 1) throw error;

      const backoff = delay * Math.pow(2, i);
      await new Promise((resolve) => setTimeout(resolve, backoff));
    }
  }
  throw new Error('Max retries exceeded');
}
```

## 安全性

### JWT 认证

```typescript
import jwt from 'jsonwebtoken';

// 生成 token
function generateToken(userId: string) {
  return jwt.sign({ userId, scope: 'user' }, process.env.JWT_SECRET!, {
    expiresIn: '1h',
  });
}

// 验证 token
function verifyToken(token: string) {
  try {
    return jwt.verify(token, process.env.JWT_SECRET!);
  } catch (error) {
    throw new Error('Invalid token');
  }
}

// 中间件
function authMiddleware(req, res, next) {
  const token = req.headers.authorization?.split(' ')[1];

  if (!token) {
    return res.status(401).json({ error: 'No token provided' });
  }

  try {
    req.user = verifyToken(token);
    next();
  } catch (error) {
    res.status(401).json({ error: 'Invalid token' });
  }
}
```

## 性能优化

### 缓存策略

```typescript
// Redis 缓存
import Redis from 'ioredis';

const redis = new Redis({
  host: 'redis-service',
  port: 6379,
});

async function getUserWithCache(userId: string) {
  // 先查缓存
  const cached = await redis.get(`user:${userId}`);
  if (cached) {
    return JSON.parse(cached);
  }

  // 缓存未命中，查询数据库
  const user = await userRepository.findById(userId);

  // 写入缓存，有效期 1 小时
  await redis.setex(`user:${userId}`, 3600, JSON.stringify(user));

  return user;
}
```

### 数据库读写分离


## 总结

微服务架构不是银弹，它带来了灵活性和可扩展性，但也增加了系统的复杂度。关键是要：

- 合理拆分服务边界
- 选择合适的通信方式
- 实施完善的监控和追踪
- 做好容错和弹性设计
- 持续优化和演进

使用 Sealos 这样的云操作系统，可以大大简化微服务的部署和运维工作。
